用成本最低的方式解决hexo博客部署在Gihub_Pages在国内访问慢的问题

前言

相信不少小伙伴都在使用 GitHub Pages 提供的免费静态网站托管来搭建自己的博客，但是因为特殊原因，使用 GitHub Pages 搭建的博客在国内访问速度极慢，因此很多小伙伴只能转战国内付费的服务器和 CDN。

然而，国内的CDN都需要域名先备案，不然无法使用。使用CloudFlare的CDN服务在国内的访问速度也并不理想。终于，我找到了相对完美的解决方案，话不多说先上图：

博客地址：http://blog.ningh.net 欢迎各位小伙伴测速

原理很简单，笔者使用了一台香港的云服务器，实现了简单的流量转发和缓存。并且使用免费的picx图床解决了Fluid主题用Typora插入图片时无法渲染的问题。实测访问速度极快。对原理不清楚的小伙伴请百度搜索 ”正向代理“。

肯定有小伙伴要说，你都有香港云服务器了，你直接把博客部署到服务器上不就得?在这脱裤子放屁干嘛？原因很简单，之前购买云服务器的时候并没有想搭建博客，所以购买的配置非常低。我购买的是阿里云的香港云服务器，公网IP，1核CPU + 0.5G（实际上只有459MB）内存 + 20G硬盘的配置，每月200G的公网流量，五年只需270元，平均才4.2一个月。我觉得这样的配置无法支撑我的博客稳定运行五年，更何况在上面我还搭建了其他服务。

不知道活动还有没有过期，这里附上服务器购买教程：https://www.cnblogs.com/tolvip/p/18269541

我们只需要准备一台海外云服务器（最好是香港，速度最快）和一个域名，接下来话不多说直接开始。

解决Fluid主题用Typora插入图片时无法渲染的问题

这一步不是必须的，不介意的小伙伴可以直接跳过这一步。

最佳的解决方案是图床。我使用picx搭建图床，图片全部保存在Github仓库，然后使用jsDelivr CDN加速访问。picx搭建图床非常简单，傻瓜式操作，具体请参考PicX使用手册。

使用jsDelivr加速也非常简单，只需要在图床设置中将图片链接规则切换为jsDelivr即可。

这样操作完，其实在大陆已经可以成功访问图片了，但是速度还是不够快。

cdn.jsdelivr.net在大陆节点很少，ping cdn.jsdelivr.net在大陆平均延迟450ms，实际下载一张图片可能延迟是2倍不止，这对读者是无法接受的。

正好cdn.jsdelivr.net在香港有节点，我用我的服务器ping延迟2ms，嘿嘿！

所以我们可以用服务器代理一下，当用户请求一张图片时，先把请求发给我的服务器，然后我的服务器再去请求cdn.jsdelivr.net，实测大陆下载一张图片只需0.1s以内，快了十倍！

我们先配置PicX链接，流量中转的代码我后面会给出，小伙伴们只需复制粘贴修改即可。在图床设置-图片链接规则配置 中新建一条规则，添加并且将图片链接规则切换为你刚刚添加的规则即可。

接下来就是配置代理服务器了。在进行下一步之前，你需要：

  1. 将域名解析到你的服务器

    举例：

    我使用blog.ningh.net中转博客流量，使用picx.ningh.net中转图床流量，这两个域名都解析到我的服务器。如果你不需要图床，那么配置一条记录即可。

  2. 服务器安装Python3环境
